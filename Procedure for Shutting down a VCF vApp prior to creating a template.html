vApp Mastering Procedure - Part 2

You must now shut down the following infrastructure VMs in the order listed below.  If your vApp does not have a particular function, then you can skip over it.

  • Tanzu Supervisor Cluster
  • vCLS
  • NSX Edge Cluster
  • NSX Manager

Follow the procedure below

  1. Click between the esxi host tabs to find the required VM(s)
  2. Select the VM
  3. Click the Shut Down button

Repeat for each instance of the VM as needed.

Repeat the above procedure for each VM in the list until all of them are shut down.

vApp Mastering Procedure - Part 3

Now that all the VMs are shut down, we can turn our attention. to modifying the labstartup files.  It is necessary to configure the labstartup procedure with the correct location of key VMs

  1. Click the Visual Studio Code Icon in the Lab Desktop
  2. Click File and then Open File
  3. Navigate to c:\hol\startup
  4. Click vSphere.ps1
  5. Click Open
  1. Click between the esxi host tabs in the web broswer, and locate the NSX manager VM (nsx-mgmt-1)
  2. Click back to Visual Studio Code and Locate the Entry that is commented # NSX Manager due to Tanzu
  3. Check that the Connect-vCenter, Start-Nested and Disconnect-VIServer entries are accurate for the NSX Manager VM
  4. Perform the same steps (6 - 8) for the other VMs in the PS1 file

When you have mande any necessary changes, you can save and close the file.

vApp Mastering Procedure - Part 4

Once the file has been updated, it is time to quiesce the disks on each esxi mgmt hosts.

  1. Click the PowerShell icon in the lab desktop
  2. Change the directory to c:\hol
  3. Type ./plog.ps1 and press enter ( Note: You may more than 4 hosts listed as a series of 1's.  That will be the plog function executing on any additional hosts beyond the four management hosts in the build).
  4. Switch back to the web browser, and on each host, change the view to Monitor and the resource to Disk

Monitor each host until the Read / Write rates flatline.  this takes about 30 minutes or so.

When all four hosts are plogged, switch back to the PowerShell prompt.

Type unplog.ps1 and press enter.
