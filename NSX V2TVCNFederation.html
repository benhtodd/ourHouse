1. vCloud Director vAppTemplate / vPod

The vAppTemplate you wish to use for delivery should be in the Livefire-Delivery-Catalog as per:

How To: Copy vAppTemplate to Delivery Org

1.1. Set V2T / VCN / Federation Class Template (if necessary)

NOTE: This is ONLY required if you have just updated your vAppTemplate

This will provide a dropdown list of the available vAppTemplate in the Livefire Delivery Org. The NSX Instructor can then choose which one should be used for upcoming deliveries. That value will be used by the Deploy V2T Class ass an input.

NOTE: The process for V2T, VCN, and NSXT-Federation vPod deployments is the same. The big difference is that the V2T request deploys 2 vPods per attendee while the others deploy 1 vPod per attendee.

Use the Livefire Automation Portal - Service Broker (Login with your HaaS Lab User account)

The page should prompt you for login... if there is a dropdown for domain selection, choose "lab.livefire.dev" - future visits to the page should show like my screenshot above.

Since the domain is already selected, you just specify your username and password then click Sign In

1.1.1. Update Delivery Template List

PLEASE NOTE: This step is ONLY required if you have JUST NOW copied a NEW template to the Delivery Catalog. This workflow is run automatically on a DAILY basis. So, if you have a new Template you want to test in the Delivery Org right away, you may need to run this simple worklow.

Request the GetandStoreDeliveryTemplates catalog item. This workflow will update the available templates in the Set VCN Template catalog item.

Click SUBMIT to run the request. It will only take a minute or two to complete.

END optional Update Delivery Template list section... you may continue on with Setting the Template for the desired class.

Request the "Set V2T Class Template" catalog item

  1. You can always see what the currently selected template for the V2T class is by simply loading this form
  2. The vAppTemplate to Use dropdown list:
    1. Is a list of the available vAppTemplates in the Livefire-Delivery-Catalog as of Midnight
    2. The list t is updated every day at midnight - via a scheduled workflow

If you have a NEW template that is to be used in an upcoming/future class and it already appears in the dropdown list, select it and click SUBMIT

If you have a NEW template that was added to the Livefire-Delivery-Catalog more than a day ago AND it does NOT show in the dropdown, please notify Burke

1.2. Deploy V2T Class

The steps below can be done up to 1 day prior to delivery.

The Workflow run-time for 25 attendees is approximately 3 Hours and 45 Minutes

This custom workflow has been written to deploy a full set of class vApps for the V2T Class. A “set” means 2 vApps per attendee.

The template used by this request is the one set by the Set V2T Class Template request in the previous step of this manual

A V2T-A-### and a V2T-B-### vApp:

The V2T-A-### vApp is immediately deployed and powered on, while the V2T-B-### is only deployed (not powered on). Once all sets for the specified number of attendees has been deployed, an e-mail is sent to the Instructor e-mail addresses that the submitter entered in the form. That e-mail provides a few helpful links as well as all the nsxuser### accounts, current attendee password at time of workflow execution, and the day 1 (V2T-A pod) and day 2  (V2T-B pod) External IP Addresses that the attendees will use to perform the labs.

When you are ready to deploy a V2T class set of pods (for testing or for actual delivery), you should use the Deploy V2T Class catalog item in the  Livefire Automation Portal - Service Broker (Login with your HaaS Lab User account)

When the form loads up, most of the default values can be left in place, but the following SHOULD be changed as appropriate for your delivery:

  1. How many attendees - this will default to 10, but can be lowered for testing purposes or increased to 25 for a full class delivery
  2. Instructor E-Mail Addresses - This is required. The addresses specified will receive an e-mail notification with the NSX Attendee Usernames (IE: nsxuser001 - nsxuser025), the currently set password for the attendee account, and the Day 1 and Day 2 Pod IP Addresses and names

During delivery, it is the instructor's responsibility to Power Down the V2T-A-### apps as the users move on to the day 2 of the class. Once the V2T-A-### app is powered off, the corresponding V2T-B-### vApp should be powered on. Set about a 15 minute expectation for the attendee.

2. Horizon Windows Desktop

NSX Attendees will login to a floating assignment Windows 10 pool as shown in the screenshot above.

When a user logs out of their desktop, that desktop is deleted. Upon logging back in, the user gets a new desktop from the pool. The pool is configured to automatically maintain a minimum number of desktops as well as spares.

As of this article, the pool

  • maintains 5 "spare" machines
  • has a maximum of 25 machines
  • auto-logs off 2 hours after disconnect (resulting in desktop deletion)
