The Lab Checkers vApp

You can create your own Lab Checkers vApp by creating one from the template called "Lab Checkers vApp". This vApp can be found in the Development Org VCD.

  1. Log in to the Development Org VCD
  2. Click on Libraries
  3. Click on vApp Templates
  4. Click on the Filter icon
  5. Type in "Checkers" in the search field

Deploy the new vApp, look up the IP address as you are used to with all the other vApps.

Us the vAPP IP address to RDP into the Control Center of the Lab Checkers vApp. Make sure you RDP using port 3390 (and NOT 3389).

When you are logged you will see the following desktop.

Preparing the scripts

Before you run the scripts, you first need to modify the CSV file (Students-v6.csv) with the Student vApp IP addresses. This list can be retrieved from the Budibase Class Roster tab. (Later in this document I will explain hor to find this inforation in Budibase).

To modify the Students-v6.csv file open Microsoft Visual Studio Code.

When Microsoft Visual Studio Code is opened click on the PSREMOTE folder in the explorer.

This will open the content of the bookmarked folder.

The scripts are all stored in: C:\hol\PSRemote\ on the Lab Checkers vApp Hard Disk.

Pay attention to the following files:

  • Students-v6.csv
    • This CVS file contains the Student vApp names and IP addresses (of the PODS).
  • CheckMultiplePods-v6.ps1
    • This script will make sure that the Lab-CNS-Full-v6.ps1script will run across all Student vAPP sthat are provided in the Students-v6.csv file.
    • This script will make sure that the JSON output (scoring information) that is retrieved from the Student vAPPs will be sent to the Budibase Database.
  • Lab-CNS-Full-v6.ps1
    • This script contains the actual Lab Task Checking and Grading where a PowerShell script will invoke several different API calls towards several different VMware Products (NSX, Manager, vCenter Server, AVI Controller, etc.) that are present in eachStudent vApp.
    • Once the script has fully been executed, it will place all the information on a perStudent vApp basis into the Budibase Database.

All other files are either old, or used for testing purpose.

Right click on the files and select "Open to the side" to open each of the files.

  1. The most important file to edit is the Students-v6.csv file.
  2. Update the vApp IP addresses and the vApp Names here.

Make sure you save the Students-v6.csv file before you run the other script.

Running the scripts

Open the Microsoft Windows PowerShell 7 Command Window (Not use version 6, but use the version 7 -> black icon).

Change the directory to C:\hol\PSRemote.

<p>cd C:\hol\PSRemote</p>
Click to copy

Execute the CheckMultiplePods-v6.ps1 PowerShell script.

<p>.\CheckMultiplePods-v6.ps1</p>
Click to copy

If the Student vApps are up and running and the information in the Students-v6.csv file is correct it will run the script. The script will iterate trough all the Student vApp IP addresses and will check the Students vApps one by one. It will take around one minute per vApp IP (POD) to complete the check, and when this is finished it will move on the next IP address in the list until it has reached the last one has finished.

I have not yet automated the script to run automatically, so if you want the latest information on the Lab Progress you need to run the script multiple times. I suggest running it at the end of the day once to get an overview of the Lab Progress of the students.

The Prerequisites

Before you run the CheckMultiplePods-v6.ps1 PowerShell script the following prerequisites need to be met:

  • The class needs to be deployed with VLP (ask Burke, Naina or Deyan for help here).
  • The Students-v6.csv file needs to be updated.
  • All Control Centers need to run Microsoft Windows Server 2022 Standard (to make sure PSRemoting is working correctly).
    • This is typically already done if you use the last CNS vAPP.
  • TCP Port 5985 and 5986 needs to be opened between the Lab Checkers vApp and the Student vApps.
    • Inside the VCD Network environment between the vApps there is firewall.

Working with the Dashboards

In order to view the data of the class in Budibase you need to access the following link:

<p>https://bb.livefire.dev/app/lab-progress#/classes</p>
Click to copy

Click on "Log in with WorkspaceONE" to log in using your SSO Livefire credentials.

Log in with your using your SSO Livefire credentials.

When a class has been provisioned (vApps have been created) with VLP you will see this in the "Active Classes".

Click on the active class to look at the Student Summary View.

When the class is no longer active you can click on Deactivate class to move it to the "Old Classes" section.

In the Student Summary View you can view each student and you will see a pie chart showing the students progress.

Click on the Student Name to drill down and review in detail what tasks have been configured (correctly) or what not.

When a student did not attend the class you can remove him from the overview by clicking the "Student not attended" button.

This is how the Dashboard looks like after you hace clicked on a Students name.

When you are in "Class View" and scroll all the way to the bottom you can see some additional buttons.

When you select "Detailed View" you can see in one overview what labs have been completed and what not on a per user basis.

When you select "View Class Roster" you can see in the Students First. Last name and the assigned Student vApp IP address.

When you select "Detailed Class Roster" you can see in the Students First, Last name, the assigned Student vApp IP address and the VLM vApp Name.

You can use this information to edit the Students-v6.csv file.
