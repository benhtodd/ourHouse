This is an automated solution to prepare for a delivery, with the help of vRA catalog you can deploy a new template in Delivery/Delivery On-demand Org and also update the respective deployment pool in VLP.

This Article explains about below catalogs:

1.  Template Copy : This catalog is useful for capturing vApp from development org and return a vApp template in selected delivery org (on-demand/delivery) with attached delivery network.

2. Update VLP Pool : This Catalog Sync lab templates in VLP with the vCD templates and updates selected deployment pool.

3. Deploy new template : This catalog is a combination of "Template copy" and "Update VLP pool" and it  executes all the steps mentioned in the above two catalogs.

Rename your development vApp with the same name you want to store your template.

As per this solution vRO takes approx 15 minutes to sync newly deployed/Updated vApps, if you can't see your vApp in the list, you must wait for sometime and try again.

Login to vRA

a. Log in to vRA using Lab Username (domain should be lab.livefire.dev) and password.

b. Select Service Broker from Livefire Automation Portal - Cloud Services Console.

Catalog : Template Copy

 To capture vApp from development org and return a vApp template in selected delivery org (on-demand/delivery) with attached network.

Steps included as part of this catalog:

  • Search for selected vApp in development org
  • Captures a temporary vApp template in development-consolidate org
  • Clone the temporary vApp template in selected destination vcd
  • Deploys a temporary vApp from the template
  • Attach external network with the temporary vApp
  • Captures the final template with the same name as the vApp

After successfully capturing the required template it deletes temporary template from selected delivery org, development consolidate org, and also deletes the temporary vApp.

Request for Template Copy catalog from the list of items, if you can't see the catalog move to next page or use the search bar.

This catalog has two inputs :

1. Select vApp Name : from the drop down list of development vApps.

Rename your vApp with the same name you want to store your template.

2. Select Destination vCD: where you want to copy your template to either Livefire-Delivery Org or Livefire-Delivery-OnDemand Org.

Click on submit, after providing correct inputs.

This process takes time, as it includes multiple subprocess.

Monitor progress of your request in Resources tab and after successful completion, verify your delivery ready template.
In case of failure contact HaaS team or follow manual approach.

How To: Copy vAppTemplate to Delivery Org

Catalog : Update VLP Pool

This catalog Syncs all (Delivery/Delivery-OnDemand) vApp templates in VLP. It updates the selected Deployment Pool with the selected vApp template and deletes existing template from the deployment pool.

Click on request. It display's vApp templates based on selected vCD Delivery Org.

Based on selected org, it shows list of template names. Select your template which should be updated to VLP deployment pool.

Select your deployment pool which is part of VLP class deployment.

After submitting the request, monitor the progress in resource tab and verify the details.

Catalog : Deploy New Template

This catalog is a combination of "Template copy" and "Update VLP pool", as it executes all the steps mentioned in the above two catalogs.

Select vApp from the drop-down list, which you will be using for the upcoming delivery.

Select vCD Destination where you want to provision your vApps.

Select VLP deployment pool where you want to map your vApp template.

After cross verifying your inputs click on submit.

Monitor the progress of this request in the resource tab. Verify the availability of your vApp template in the destination vCD. Also validate for updated template in VLP deployment pool.

In case of any concern, please contact Haas Team.
