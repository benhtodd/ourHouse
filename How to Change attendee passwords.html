We are using vRealize Suite for overall Lab Environment Management and Automation. This article will step you through the process of logging in and changing a team's attendee account.

The Password change workflow has been altered to allow for changing just the first 50 user accounts (most commonly used for any/all deliveries) and storing that password in a text file on a web server for easy retrieval. This allows for user accounts 051-100 to be used by instructors and/or Attendees/Take3 members who require additional use of our labs.

 As of this article, the following are available for automated change:

Select the below options based on requirement:

  1. XXXuser001-050:  Default run, resets password for range "md/is/nsx/mi/cs/dw/user 001-050" (This is the only option that updates the password that is retrieved from workflows and/or url: http://vcd-util.livefire.lab/XXXuser (where XXX is cs, dw, nsx, mi, md, or is)
  2. Specific User: Resets password for selected user (XXXuser###) from the displayed user list
  3. Multiple Users: Resets password for multiple selected users at once from the list of displayed users

Accordingly catalogs are available for the below teams:

  • Cloud Services Attendees (csuser###),
  • Digital Workspace Attendees (dwuser###),
  • Modern App Infrastructure Attendees (miuser###),
  • Moder App developer Attendees (mduser###),
  • Intrinsic Security Attendees (isuser###),
  • NSX Attendees (nsxuser###)

If you wish to reset ALL passwords to match the password that gets stored in the url, then run option 1 AND 3.

1. Log in to our Livefire Automation Portal Catalog

  1. Log in to https://vra.lab.livefire.dev/catalog using your Lab Username (domain should be lab.livefire.dev) and password

2. Select Service Broker if not already there

  1. If you didn't include /catalog in the url or you get to this portal page, click on Service Broker

3. Request Change-XXX-Password

Depending on your Team Membership and what Filters you have set, you may see different items in the catalog than what is shown here in the current Cloud Service Team Catalog. If you happen to see multiple "Change XXXuser Pass..." entries, be very sure to select the one for YOUR team only when coming in to change attendee passwords.

  1. Click REQUEST for the Change Password of Catalog item for your team...

4. Set New Password and Select Users option from the drop down "User Selection"

This catalog can be used to reset password for user range from 1 to 50, a single user or multiple users based on the selected options mentioned below:

  • User001-050:  Default run, resets password for user range from "001 to 050", click submit and it updates the password for all the users within 001 to 050 range.

NOTE: The Last Set Password is only set/updated when this option (User001-050) is selected!

Specific User:

  • Select Specific User option from User Selection dropdown, it adds another field in the input form as Select Specific User, which displays all the available users.
  • Select  a specific user csuser### from the available users list displayed under Select Specific User.
  • Click submit and it resets password for selected user.

Multiple Users: 

  • Select Multiple Users option from the User Selection dropdown, it displays another field as Select Multiple Users, which displays all the available users.
  •  Select multiple users for which password needs to be reset from the users list under Select Multiple Users.
  • Click submit and it resets password for all selected users at once.

5. Deployment Completes - Password is ready

You can verify the new password has been set by checking the current password for your team's attendee account (REMINDER: This password check is ONLY applicable to the user001-050 option), then test login to a Horizon Desktop with that account.
