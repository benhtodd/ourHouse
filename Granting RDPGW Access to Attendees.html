We are using vRealize Suite for overall Lab Environment Management and Automation. This article will step you through the process of granting RDP gateway access to attendees via dedicated RDP Gateway which is restricted to specific IPs.

Grant RDPGW Access catalogs are available for all the teams like : NSX Users access to the external IPs of their Pods, IS Users access to the external IPs of their pods, and DW Users access to the external IPs of their Pods, to provide RDP gateway access, please select the user from the list, enter external POD IPs and the email address in which details needs to be sent, by default itâ€™s granting access for all the respective ports : 3389,3390,3391,3392,3393,3394,3395

Unfortunately, the RDP GW does not let you simply specify a range of IPs or CIDR, it is list of Individual IP Addresses, only starting with "172.25.X.X"

Performance is quite a bit slower than a Horizon Desktop so Horizon should always be your preferred entry-point to the Lab Environment. This solution is simply an alternative/backup solution for when there are issues with Horizon.

1. Log in to Livefire Automation Portal Catalog

  1. Log in to https://vra.lab.livefire.dev/catalog using lab Username (domain should be lab.livefire.dev) and password.
  2. Select Service Broker from Livefire Automation Portal - Cloud Services Console

2. Request Grant RDPGW Access Catalog

Select the Grant RDPGW Access catalog item, based on respective teams. The below icon is used to identify RDPGW Catalogs:

 

Catalog names and it's respective teams are listed below:

  • Grant RDPGW Access - NSX Users
  • Grant RDPGW Access - IS Users
  • Grant RDPGW Access - DW Users

After selecting the correct catalog, click on request.

For the below example Grant RDPGW Access - NSX Users is used.

Provide the below input details like:

  • select the user for which RDPGW access to be granted
  • mention the IPs
  • e-mail in which login details needs to be sent and the access revocation date

Select User:  select attendee username from the available list of users.

for this deployment nsxuser099 is selected.

Primary IP Address: Provide Pod IP address for which RDPGW access needs to be granted. It's mandatory to provide IP in the format of "172.25.X.X", or else it will throw error saying "Invalid format".

Secondary IP Address: In case of two pods, specify the second pod ip in the secondary IP address field with the same format "172.25.X.X"

For this deployment just one pod is used so filled the ip in the Primary IP Address field.

E-mail Addresses: Provide Instructors email address in which RDP file and login related details needs to be sent, if it's more than one e-mail click on plus next to the text input and enter the address.

Revoke RDPGW Access on Date : select the RDPGW access revocation date and time from the calendar and clock. In case of no selection the workflow will remove access on 7th day.

Click on SUBMIT, after the request submission you can track the request details on the Deployments tab.

3. Access RDP Server

Once the request is granted vRO sends an e-mail from "Livefire-Haas-vRO <vro-alerts@lab.livefire.dev>" , to the address mentioned in the form while submitting the request.

As the subject of the mail explains it contains "Livefire RDP server details", download the RDP file from the attachment (Livefire Desktop.rdp), share the login details with the attendee.

In case for two POD IPs, the e-mail will show two attachments:

1. Livefire Desktop.rdp contains the details of primary pod (Primary IP Address) related details.

2. Livefire Desktop2.rdp contains the details of secondary pod (Secondary IP Address) related details.

Open the dowloaded RDP file, it will ask for attendee password, it's already shared with the attendee and instructor.

For instructions on How to Change attendee passwords, see this article:  How to Change attendee passwords

After continuing with the correct attendee password it throws another popup to provide "administrator@corp.local" password, the correct password is  "VMware1!", as displayed in the e-mail.

It starts configuring remote PC.

Confirm the certificate informations and click on continue.

In few seconds your RDP will be loaded.

The RDPGW access will be revoked as per the revocation date mentioned while submitting the request.
