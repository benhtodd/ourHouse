Attendee Password Accounts are managed by a vRO Workflow that is serviced up in our Lab's Service Broker. There are two ways to retrieve the last password that a team's attendee account has been set to. Via URL or via the Change Password Form

PLEASE REMEMBER TO RESET YOUR TEAM'S ATTENDEE PASSWORD AFTER DELIVERY

We should Only permit access to our lab environment for the duration of a delivery, changing the passwords the day (or couple days) following a delivery. Please make sure this becomes part of your regular routine.

1. URL

Whenever the workflow is run, it will connect to a web server and store the password in a text file.

This URL should be kept to Instructors only and it is ONLY accessible from within the lab environment.

1.1. Digital Workspace Team - dwuser###

From a browser: http://vcd-util.livefire.lab/dwuser

If you are an Ubuntu Instructor Desktop user and spend a lot of time at a console, then you may want to setup an alias:
Copy and paste the following lines into your Instructor Ubuntu Desktop Terminal. Future launches of your terminal will include this alias:

 

echo 'alias getdwuserpw="curl http://vcd-util.livefire.lab/dwuser"' &gt;&gt; $HOME/.zshrc
source $HOME/.zshrc
Click to copy

Now, when you type getdwuserpw, you will see the last password set by the workflow.

1.3. Cloud Services Team - csuser### / mduser### / miuser###

The cloud services team has three different user accounts (and corresponding attendee ubuntu desktop pools):

  • csuser### - legacy user account tied to the ATT-CSL-### Desktops (Pool name: ATT-UBUNTU)
  • mduser### - Modern Apps Developer User, tied to the ATT-MDL-### Desktops (Pool name: ATT-MDL)
  • miuser### - Modern Apps Infrastructure User, tied to the ATT-MIL-### Desktops (Pool name: ATT-MIL)

2. Change XXX Password Form

When you login to https://vra.lab.livefire.dev/catalog (Service Broker) and choose the catalog item to change your team's Passwords, the last set password will be read from the relevant URL and displayed on the form. Int this example, I'm on the form to change the nsxuser###@livefire.lab account password. Submitting that form will change all user account passwords found in the NSX_VDI OU in the Livefire.Lab AD Server, add the password to the "Description" Field of the AD Accounts AND will update the text file found at http://vcd-util.livefire.lab/nsxuser - This same process applies to each team's designated attendee accounts: dwuser###, csuser###, isuser###, mduser###, miuser###, and vcfuser###
