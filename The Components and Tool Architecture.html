The components

The tool consist out of multiple elements that makes up the full solution.

  • The Lab Checkers vApp
  • The Lab Checkers Scripts
  • The Student vApps
  • Budibase (Database and Dashboard)

The Lab Checkers vApp

The Lab Checkers vApp is the main control center where the LabCheckers Scripts are initiated from.

The PowerShell/PowerCLI Scripts are hosted on this vAPP and PSRemoting will be used to execute the scripts on all the Student vApps.

The Lab Checkers Scripts

This Lab Checkers Scripts consist out of 3 main files:

  • Students-v6.csv
    • This CVS file contains the Student vApp names and IP addresses (of the PODS).
  • CheckMultiplePods-v6.ps1
    • This script will make sure that the Lab-CNS-Full-v6.ps1 script will run across all Student vAPPs that are provided in the Students-v6.csv file.
    • This script will make sure that the JSON output (scoring information) that is retrieved from the Student vAPPs will be sent to the Budibase Database.
  • Lab-CNS-Full-v6.ps1
    • This script contains the actual Lab Task Checking and Grading where a PowerShell script will invoke several different API calls towards several different VMware Products (NSX, Manager, vCenter Server, AVI Controller, etc.) that are present in each Student vApp.
    • Once the script has fully been executed, it will place all the information on a per Student vApp basis into the Budibase Database.

The task can have three possible outcomes:

  • Pass
    • A Lab task has been successfully configured with no mistakes so 1 point is awarded.
  • Fail
    • A Lab task has not been successfully configured or a mistakes have been made so 0 point are awarded.
  • Not Checked
    • Sometimes tasks are configured and this configuration is deleted later in the lab (to make other labs work in the future), in this case the task will not be checked and  so 0 point are awarded.

The Student vApps

The scripts are remotely being executed from the  Lab Checkers vApp where the target will be the Student vApps defined in the Students-v6.csv file.

Each student vAPP needs to have a Control Center VM based on Microsoft Windows Server 2022 Standard. The windows version is important because the older versions will not work well with PSRemoting,

Each student vApp will have the full lab deployed with various different VMware products that will be used in the Class. (NSX, Manager, vCenter Server, AVI Controller, etc.)

Budibase

Budibase is a tool that is considered as a NoCode or LowCode tool to create advanced application tailored to your needs.  Budibase works together with the VLP system to provision PODS (Student vApps) for each class. (Burke, Naina and Deyan can tell you all about VLP). Budibase is used for the VLP solution and is also used in the Lab Checkers Solution.

The data that the Lab Checkers scripts retrieves from the Student vApps will be sent to Budibase, and Budibase will place this into the Budibase Database.

Once the Data is in the (Budibase) Database a front end will retrieve the data using various different queries to display the data on a (Budibase) web based front-end (the Dashboard).
