In this lab, we will create an ABX action that will post a notification message with information about your deployment in a Slack channel.

Testing the SSH Key ABX Action

Go to Design > Blueprints and import the ABX SSH Key.yaml blueprint from Dropbox . For name use ABX SSH Key - yourname . Deploy the blueprint .

Go back to the Extensibility Tab.

Navigate to Activity > Action Runs and monitor for the action to complete. 

Open the action run once it has completed. 

On the right side, scroll down to see the outputs in the Details tab. You should see that we have successfully injected the SSH Key into the payload.  

Go back to the Deployment tab and get the IP of the VM from the deployment details. We are going to need this next to SSH into the VM.

Go to your Horizon Desktop and SSH to the machine using the private key provided in the Dropbox. You can verify the public key was copied to your server by running the following commands:

First, fix a permission problem on the private SSH key:

chmod 600 ~/.ssh/id_rsa

Now, attempt to SSH to your server:

ssh centos@[your-server-IP]
