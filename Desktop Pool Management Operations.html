We are using vRealize Suite for overall Lab Environment Management and Automation. This article will step you through the process of logging in and managing Horizon Desktop Pools i.e Recover, Reset, or Restart any VM in the pool OR Recover the entire pool back to base image.

Desktop Pool Management catalogs are available for the teams using Linux based VMs for the Attendees Labs. Catalog names and it's respective teams are listed below:

  • ModernApp Dev Desktops Catalog is for Modern Applications Livefire : Cloud Native Developer
  • ModernApp Infra Desktops Catalog is for Modern Applications Livefire : Cloud Native Infrastructure
  • VCF Desktops Catalog is for VCF Livefire : Architecture & Automation
  • TAKE3 Desktops is for Take3 Users Virtual Machines
  • Instructor-Ubuntu-Desktops is for Instructor Desktops

1. Log in to our Livefire Automation Portal Catalog

  1. Log in to https://vra.lab.livefire.dev/catalog using Lab Username (domain should be lab.livefire.dev) and password.
  2. Select Service Broker from Livefire Automation Portal - Cloud Services Console

2. Request Desktop Pool Management Catalog

Select the catalog item, based on respective teams and tagged Horizon Desktop Pools.

The below icon is used to identify the Desktop Pool Management Catalogs:

Available Catalogs with the tagged Horizon Desktop Pools:

  • Modern App Infra Desktops - ATT-MIL
  • Moder App dev Desktops - ATT-MDL
  • Cloud Services Desktops - ATT-UBUNTU
  • VCF Desktops - ATT-VCFL
  • TAKE3 Desktops - ATT-TAKE3
  • Instructor-Ubuntu-Desktops - Instructor-Ubuntu-Desktops

After selecting the correct catalog, click on request.

For the below example Modern App dev Desktops is used.

3. Select the required action

The below actions are available to manage the desktop pools with their corresponding virtual machines

Note : Be cautious while selecting the below actions as proceeding will result in the loss of any saved and unsaved user data of the Virtual Machine

Available Actions:

  1. Recover Entire Pool : Recovers the entire pool back to base image.
  2. Recover Multiple Machines : Recovers the selected multiple virtual machines from the available machines of the respective pool back to base image.
  3. Recover an Individual Machine : Recovers the selected individual virtual machines from the available machines of the respective pool back to base image.
  4. Reset an individual Machine: Resets the selected individual virtual machines from the available list of machines.
  5. Restart an Individual Machine : Restarts the selected individual virtual machines from the available list of machines.

a. Recover Entire Pool

Select "Recover Entire Pool" to recover the entire pool(all the available virtual machines of the respective pool) back to base image. Aafter selection of this action, it adds a new tab next to General tab, for the desktop pool name confirmation.

Are you sure you want to go with "Recover Entire Pool", as proceeding will result in the loss of all saved and unsaved user data as well as disconnects from all active sessions, if yes click on "Confirmation" tab and provide the Desktop Pool name.

Desktop Pool Name Confirmation :  Type the correct  desktop pool name in the same format as it displayed in the General tab against "Horizon Pool Name" for which "RECOVER" action needs to be taken.

Note : Desktop Pool Name Confirmation is a mandtory field which appears only in case of entire pool recovery.

b. Recover Multiple Machines

To recover multiple virtual machines, select machine names from the displayed list of available machines.

Are you sure you want to go with Recover Virtual Machine, as proceeding will disconnect from all active sessions and loss of all saved/unsaved user data and converts back to the base image.

c. Recover an Individual Machine

To recover an individual machine, select a machine name from drop down of available machines which needs to be recovered.

d. Reset an Individual Machine

To reset an individual machine, select a machine name from drop down of available machines which needs to be reset .

Are you sure you want to go with Reset Virtual Machine, as proceeding will result in the loss of all unsaved user data as well as disconnects all active sessions.

e. Restart an Individual Machine

To restart an individual machine, select a machine name from drop of available machines which needs to be restarted .

Are you sure you want to go with Restart Virtual Machine, as proceeding will result in the loss of all unsaved user data as well as disconnects all active sessions.

4. SUBMIT the Request

After cross verifying all the selected options and it's corresponding inputs click on submit.

5. Confirmation on successful/failed deployment

The above mentioned actions take few seconds to complete and displays the result in the Deployments tab, select the appropriate deployment (catalog name plus the deployment time).

Select the deployment to verify the details and see Create Successful with the list of events performed.

To cross verify the request details select Request Details next to Events tab and confirm the Action Name, Machine Name and Horizon Pool Name etc.

Deployment Completes

Now the virtual machines should function as expected with a fresh login.
