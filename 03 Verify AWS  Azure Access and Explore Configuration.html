Exploring the AWS configuration Â 

During the labs, you will be referring back to these steps over and over again to explore the objects deployed in the cloud. The steps below outline the parts of the configuration that are of interest for the labs

All attendees are sharing the same view-only@livefire.solutions user account to access AWS and Azure consoles. Therefore please, oh pretty please, do not modify views or attempt to change anything else as you may affect the experience for everyone else in the class.

We will be working in 3 regions :

  • US East (Ohio)
  • US West (N. Carolina)
  • US West (Oregon)  

Everything you deploy from the VMware Cloud Services Portal (CSP) will land in one of those regions. All regions have exactly the same configuration. Make sure you are in the correct regions when you are looking for a particular deployment.

To switch regions click on the upper right corner

Go to Services and under Compute, select EC2

Navigate to Instances. Here you will see the deployed instances. You may not see any instances now because you have not deployed any. You will see these later on when you deploy the Tito application for the first time.

Select an instance and navigate to the Tags tab. Here you can see all the tags assigned to the instance.

Go to Services and Search for VPC then select the VPC service.

Select Subnets. Here you can see all the subnets that are synced to the Cloud Assembly inventory.

You can explore the Auto scale configuration by navigating to Auto Scaling and selecting Lunch Configuration and Auto Scaling Groups

All attendees are sharing the same view-only@livefire.solutions user account to access AWS and Azure consoles. Again please, oh pretty please, do not modify views or attempt to change anything else as you may affect the experience for everyone else in the class.

We will be working in 3 regions :

  • US East (Ohio)
  • US West (N. Carolina)
  • US West (Oregon)  

Everything you deploy from the VMware Cloud Services Portal (CSP) will land in one of those regions. All regions have exactly the same configuration.

Navigate to Resource Groups. Here you can see the three resource groups you will be deploying into during the delivery. One group per each region:

  • elasticsky-dev-rg
  • elasticsky-test-rg
  • elasticsky-prod-rg

Navigate to Virtual machines. Here you will see all the deployed virtual machines. you can also see the Tags associated with these machines.

You may not see any instances now because you have not deployed any. You will see these later on when you deploy the Tito application for the first time.

Navigate to Virtual networks. Here you can see the virtual networks which are pre-configured for the labs to work.

Select one of the networks and explore it. Navigate to Subnets. Here you can see the subnets from that network that are being synced to Cloud Assembly.

Navigate to Network Security Groups. Here you can see the virtual networks which are pre-configured for the labs to work.

Explore one of the networks. You can see inbound and outbound rules created to allow your app to be reachable from the internet.

Navigate to Disks. Here you can see the disks that are associated with deployed virtual machines.
You may not see any disks now because you have not deployed any VMs yet. You will see these later on when you deploy the Tito application for the first time.
