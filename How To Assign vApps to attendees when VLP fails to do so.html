For several weeks now we are observing failures when VLP assigns the pre-provisioned vApps (prepops) to the entitled users. This results in vApps not being renamed as expected.

This seems like a bug in VCD and the HaaS Lab Team are working with GSS on confirming that.

1. Symptoms of the problem

The following error would be seen in the VLP portal:

And in the Cloud director you could see that the vApps are not renamed reflecting each attendee's e-mail in their name:

2. The workaround

If you don't see "Cloud Tasks" you are probably using the old VLP navigation. You can switch to the new navigation at the very bottom of the pane on the left or if you prefer using the old UI, the "Cloud Tasks" are located under "Task Management"

Log in to the VLP portal and navigate to Cloud Management -> Cloud tasks. Then click on the vcd.livefire.dev cloud task

On the window that just opened, go to the Failed Tasks tab and sort them by date. The newest ones on should be the ones for your class. Select the three dots menu to the left of each task and select "re-submit". This will re-run each task and will succeed in renaming the vApps.
This needs to be done for each task. Unfortunately, there is no way of selecting multiple tasks and performing an action on all selected.

Once all the tasks succeed, you woulds see the vApp getting renamed in the VCD. After this completes, you could inspect the events for a vApp, where you would see that the second attempt for vApp modification succeeds:
