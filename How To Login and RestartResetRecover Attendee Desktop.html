Session Page Actions

Some of these actions will have different results depending on whether they are performed on Ubuntu (Persistent) or Windows (Non-Persistent) Desktops.

1. Disconnect Session

  • Can only be used with a single session selected
  • The Disconnect Session button has the same results on Persistent/Non-Persistent desktops - It simply Disconnects the session, leaving the user account logged into the desktop. The user would just need to re-connect to the desktop pool and their session would resume.
  • User may need to wait 1-2 minutes before attempting to reconnect

2. Logoff Session

  • Can be used with multiple sessions selected
  • The Logoff Session button will actually log the user account out of the desktop OS.
    • For non-persistent desktops, this typically means the desktop will then be deleted. When the user attempts to connect back in to the Desktop Pool, they will have a fresh desktop.
    • For persistent desktops, this simply means when they login, the login script (if applicable) will re-run and they will have the same VM they were connected to before.
  • User may need to wait 3-4 minutes before attempting to reconnect

3. Restart Desktop

  • Can only be used with a single session selected
  • The Restart Desktop button does a guest OS restart. The effects for the persistent vs. non-persistent are the same as Logoff Session:
    • Non-persistent desktops will be deleted and user will connect to a fresh OS
    • Persistent Desktops will simply reboot and user will re-connect to the same VM
  • User may need to wait 3-5 minutes before attempting to reconnect

4. Reset Virtual Machine

  • Can only be used with a single session selected
  • The Restart Desktop button does a hard restart (no guest OS shutdown - simply powerOff/PowerOn). The effects for the persistent vs. non-persistent are the same as Logoff Session:
    • Non-persistent desktops will be deleted and user will connect to a fresh OS
    • Persistent Desktops will simply reboot and user will re-connect to the same VM
  • User may need to wait 3-5 minutes before attempting to reconnect

5. Send Message

  • Can be used with multiple sessions selected

Machines Page Actions

If you need to set a desktop (Or set of desktops) back to a fresh image, you can't use the Session page. The Machines page is a quick and easy way to get to the "RECOVER" button. This is the button we use to Delete a "used" desktop and replace it with a fresh instant-clone.

The RECOVER process is only applicable to Ubuntu Desktops since they are Dedicated Assignment desktops that persist through logouts and restarts/reboots. The Windows desktops delete automatically upon logout or reboot.

  1. Click Machines in the left menu under Inventory
  2. Apply a filter for your desired user or desktops. In this example, I have entered MDL to show all deployed "Modern Apps Developer" desktops. Ubuntu Desktop Pools can be filtered for as follows:
    1. MDL: Modern Apps Developer
    2. MIL: Modern Apps Infrastructure
    3. VCFL: VCF Desktops
  3. Select the Desktop you wish to recover back to a clean image
  4. Click the Recover button

In 5-10 minutes, the fresh desktop will be available for login.
